# ChatSphere — Estado de fases

> Mensajería en tiempo real tipo Slack/Discord: servidores, canales públicos/privados, DMs, presencia,
> "escribiendo…", reacciones, menciones — todo en tiempo real con **SignalR + Redis**. Leer al iniciar sesión.
> Vara de calidad: **FinPulse** (NO NEGOCIABLE — PORTFOLIO_PROJECTS.md §2 "Quality gate"), incl. **Capa de Demo Guiada**.
> **Deploy diferido**: se construye completo menos el deploy en la nube (decisión del usuario, en bloque al final).

## Stack
- **Front:** Angular 20 (standalone + signals) + Tailwind v4 + RxJS. **Modo oscuro por defecto** (grafito + acento violeta `#7C6CF0`).
- **Back:** .NET 9 Web API (Clean Architecture) + **SignalR** (hubs) + EF Core.
- **DB:** SQL Server 2022 (mensajes/historial) + **Redis** (presencia + SignalR backplane, vía Docker).
- **Auth:** JWT + refresh rotativo + lockout. RBAC por servidor (Owner/Admin/Member).
- **Deploy:** Azure (SignalR Service) — diferido.

## Cómo correr (dev)
1. SQL Server local (Windows auth) en `localhost`. La API crea/migra/seedea la BD `ChatSphere` al arrancar.
2. Redis: arrancar Docker Desktop → `docker compose up -d redis`.
3. `cd backend && dotnet run --project src/ChatSphere.Api` → API en `http://localhost:5191` (`/health`).
4. `cd frontend && npx ng serve` → app en `http://localhost:4200` (login con cuenta demo).
